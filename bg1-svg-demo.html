BG1 SVG 矢量背景 - 游戏集成演示

对比 SVG 原生渲染 vs Canvas drawImage 加载 vs CSS 背景，给出最优集成方案

方案 A: SVG 原生嵌入 画质最佳

直接用 <img> 或 <svg> 标签嵌入。滤镜、渐变完全保真。适合菜单界面、关卡选择等静态展示场景。

方案 B: Canvas drawImage 加载 SVG 性能均衡

将 SVG 作为 Image 对象加载，每帧 drawImage 绘制。可缩放、可叠加游戏元素。注意：复杂 SVG filter 在某些浏览器 Canvas 中可能降级。

方案 C: CSS 背景 + 透明 Canvas 性能最佳

SVG 作为容器 CSS 背景，Canvas 设透明只绘制游戏元素。GPU 分层渲染，性能最好，但 Canvas 滤镜（screen 混合模式）可能与背景产生意料外混合。

方案 D: Offscreen Canvas 预渲染 游戏最优

游戏启动时将 SVG 绘制到隐藏的 offscreen canvas，之后每帧直接 drawImage(offscreen)。避免了每帧解析 SVG 的开销，是游戏中背景的最佳实践。

集成到现有游戏代码（game.js）

// game.js - 在 Game 构造函数中添加背景初始化
constructor() {
    // ... 原有代码 ...
    
    // === SVG 背景预渲染 ===
    this.bgImage = new Image();
    this.bgImage.src = 'assets/bg1-vector.svg';
    this.bgLoaded = false;
    this.bgImage.onload = () => {
        this.bgLoaded = true;
        // 预渲染到 offscreen canvas，避免每帧重绘 SVG
        this.bgCanvas = document.createElement('canvas');
        this.bgCanvas.width = CONFIG.CANVAS_WIDTH;
        this.bgCanvas.height = CONFIG.CANVAS_HEIGHT;
        const bgCtx = this.bgCanvas.getContext('2d');
        // 保持比例铺满
        const scale = Math.max(
            CONFIG.CANVAS_WIDTH / this.bgImage.naturalWidth,
            CONFIG.CANVAS_HEIGHT / this.bgImage.naturalHeight
        );
        const drawW = this.bgImage.naturalWidth * scale;
        const drawH = this.bgImage.naturalHeight * scale;
        const offsetX = (CONFIG.CANVAS_WIDTH - drawW) / 2;
        const offsetY = (CONFIG.CANVAS_HEIGHT - drawH) / 2;
        bgCtx.drawImage(this.bgImage, offsetX, offsetY, drawW, drawH);
    };
}

draw() {
    // 替换原有的纯色背景填充
    // ctx.fillStyle = '#2d2d3a';
    // ctx.fillRect(0, 0, CONFIG.CANVAS_WIDTH, CONFIG.CANVAS_HEIGHT);
    
    if (this.bgCanvas) {
        ctx.drawImage(this.bgCanvas, 0, 0);
    } else {
        // 后备：纯色
        ctx.fillStyle = '#2d2d3a';
        ctx.fillRect(0, 0, CONFIG.CANVAS_WIDTH, CONFIG.CANVAS_HEIGHT);
    }
    
    this.drawGrid();
    // ... 后续绘制代码不变 ...
}
// game.js - 简单 drawImage 方式（无需预渲染）
constructor() {
    this.bgImage = new Image();
    this.bgImage.src = 'assets/bg1-vector.svg';
    this.bgLoaded = false;
    this.bgImage.onload = () => { this.bgLoaded = true; };
}

draw() {
    if (this.bgLoaded && this.bgImage.complete) {
        // cover 模式：保持比例铺满
        const scale = Math.max(
            CONFIG.CANVAS_WIDTH / this.bgImage.naturalWidth,
            CONFIG.CANVAS_HEIGHT / this.bgImage.naturalHeight
        );
        const w = this.bgImage.naturalWidth * scale;
        const h = this.bgImage.naturalHeight * scale;
        ctx.drawImage(this.bgImage, 
            (CONFIG.CANVAS_WIDTH - w) / 2, 
            (CONFIG.CANVAS_HEIGHT - h) / 2, w, h);
    } else {
        ctx.fillStyle = '#2d2d3a';
        ctx.fillRect(0, 0, CONFIG.CANVAS_WIDTH, CONFIG.CANVAS_HEIGHT);
    }
    // ...
}
// index.html - 将 Canvas 改为透明，SVG 作为 CSS 背景
<div id="gameContainer" style="
    position: relative;
    background: url('assets/bg1-vector.svg') center/cover no-repeat;
    width: fit-content;
">
    <canvas id="gameCanvas" style="background: transparent;"></canvas>
</div>

关键注意事项

1. SVG filter 兼容性：<filter> 中的 feGaussianBlur 在 Canvas drawImage 时大部分浏览器支持良好，但某些复杂滤镜组合可能降级。如果遇到问题，可在 SVG 中简化 filter 或改用 CSS filter。

2. 文件大小：当前 SVG 约 4KB（gzip 后约 1.5KB），相比 PNG 大幅减小，且任意缩放不模糊。

3. 动态修改：SVG 内联到 DOM 后可用 JS 实时修改颜色、位置（如昼夜切换、水母游动）。若用 img/src 方式加载则无法修改。

4. 加载时机：SVG 作为 Image 加载时，必须等 onload 后再 drawImage，否则可能绘制空白。务必保留纯色后备填充。
