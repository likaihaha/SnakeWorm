# SnakeWorm 贪吃蚯蚓 — 分阶段原型开发计划

> 基于 `SnakeWormDesign.md` 设计文档，采用**原型驱动开发**策略，从最简单可玩版本开始，逐步叠加核心机制，每阶段交付后根据反馈调整再进入下一阶段。

---

## 技术选型

| 项目 | 选择 | 理由 |
|------|------|------|
| 引擎 | **HTML5 Canvas + 原生 JavaScript** | 零依赖、即开即玩、无需构建工具、便于快速迭代 |
| 渲染 | Canvas 2D API | 2D 像素/方块风格足够，性能可控 |
| 架构 | 模块化 ES6 Class | 每个核心机制独立模块，便于替换和扩展 |
| 交付 | 单 HTML 文件（内嵌 CSS/JS） | 每阶段一个独立文件，方便对比和回滚 |

---

## 各阶段详细规划

### 🔹 阶段 1：v0.1 — 基础移动与生长

**目标：** 让玩家能控制一条蚯蚓在 2D 平面上移动，吃食物后身体变长。验证最基础的操控手感和分段身体渲染。

**包含内容：**
- [ ] 游戏画布（800x600，泥土背景色）
- [ ] 蚯蚓数据结构：头部 + 身体段数组，每段记录坐标
- [ ] 玩家控制：WASD / 方向键 / 鼠标指向移动
- [ ] 移动逻辑：头部按方向移动，身体段跟随（平滑跟随或逐格跟随）
- [ ] 食物生成：随机位置生成"泥土块"（小方块）
- [ ] 碰撞检测：头部与食物碰撞 → 身体增长（尾部追加段）
- [ ] 边界处理：撞墙则游戏结束（或穿墙，暂定撞墙结束）
- [ ] 基础 UI：显示当前长度（分数）

**暂不包含：** 自噬、分裂、松土、AI、道具、音效

**验收标准：**
- 蚯蚓可流畅操控，无明显卡顿
- 吃食物后长度正确增加
- 撞墙游戏结束，显示最终长度

---

### 🔹 阶段 2：v0.2 — 自噬与分裂

**目标：** 实现核心创新机制——自噬不结束游戏，而是分裂为新蚯蚓。这是与传统贪吃蛇最大的区别。

**包含内容：**
- [ ] 自噬检测：头部与自身身体段碰撞检测
- [ ] 分裂逻辑：
  - 碰撞点后的身体断裂
  - 断裂部分在原地"抖动"几秒
  - 抖动结束后，断裂段头部生成新头部，成为独立蚯蚓
  - 新蚯蚓颜色随机（红/蓝/绿），继承原长度 30%
- [ ] 新蚯蚓基础移动：随机游走（简单 AI）
- [ ] 多蚯蚓管理：数组管理所有蚯蚓（玩家 + 分裂体）
- [ ] 碰撞扩展：蚯蚓头部与其他蚯蚓身体碰撞 → 被吞噬（游戏结束或对方增长）
- [ ] 分裂体数量上限：防止无限分裂导致卡顿

**验收标准：**
- 自噬时游戏不结束，正确触发分裂
- 新蚯蚓正确生成并独立移动
- 头部被其他蚯蚓吞噬时游戏结束

---

### 🔹 阶段 3：v0.3 — 松土与加速

**目标：** 实现"松土"机制——蚯蚓经过的路径变为"泥洞"，在泥洞中移动速度加快。

**包含内容：**
- [ ] 地图网格化：将画布划分为网格，每个格子记录"泥土"或"泥洞"状态
- [ ] 松土逻辑：蚯蚓经过的格子变为泥洞（视觉变透明或变色）
- [ ] 加速逻辑：蚯蚓头部在泥洞格子上时，速度 +50%
- [ ] 松土范围衰减：蚯蚓越长，松土范围越小（或松土持续时间越短）
- [ ] 视觉反馈：泥洞用不同颜色/透明度表示
- [ ] 泥洞恢复（可选）：泥洞一段时间后恢复为泥土

**验收标准：**
- 蚯蚓经过后正确留下泥洞
- 在泥洞中移动明显加速
- 长蚯蚓的松土范围正确衰减

---

### 🔹 阶段 4：v0.4 — AI 智能体

**目标：** 让分裂体 AI 更智能，能追逐食物、避开危险、利用泥洞加速。

**包含内容：**
- [ ] AI 行为树/状态机：
  - **觅食状态**：向最近食物移动
  - **避敌状态**：检测附近其他蚯蚓头部，远离
  - **加速状态**：优先在泥洞中移动
  - **攻击状态**（后期）：尝试包围玩家
- [ ] 感知系统：AI 视野范围，检测食物/敌人/泥洞
- [ ] 难度递增：随游戏时间，AI 攻击性增强
- [ ] AI 自噬：AI 蚯蚓也会自噬分裂
- [ ] 性能控制：AI 计算量控制，避免卡顿

**验收标准：**
- AI 能主动寻找食物并增长
- AI 会避开玩家头部（不主动送死）
- AI 会利用泥洞加速
- 多 AI 同时运行不卡顿

---

### 🔹 阶段 5：v0.5 — 道具系统

**目标：** 添加随机道具，增加游戏变数和策略深度。

**包含内容：**
- [ ] 道具生成：随机位置、随机类型生成道具
- [ ] 加速光珠：
  - 外观：发光黄色圆点
  - 效果：吞噬后速度大幅提升（+100%），持续 5 秒
- [ ] 晶核：
  - 外观：蓝色菱形
  - 效果：吞噬后获得一次"免疫自噬"护盾（下次自噬不分裂）
- [ ] 道具持续时间：效果结束后恢复正常
- [ ] 道具重生：道具被吞噬后，随机时间后在随机位置生成新道具
- [ ] 视觉反馈：道具闪烁、吞噬特效、状态图标

**验收标准：**
- 道具正确生成和显示
- 吞噬后效果正确触发和持续
- 效果结束后正确恢复

---

### 🔹 阶段 6：v1.0 — 完整游戏

**目标：** 整合所有机制，添加 UI、音效、视觉优化，形成可发布的完整游戏。

**包含内容：**
- [ ] 开始菜单：游戏标题、开始按钮、操作说明
- [ ] 游戏 HUD：
  - 当前长度 / 最高分
  - 速度计
  - 道具状态图标
  - 分裂体数量
- [ ] 游戏结束画面：最终分数、重新开始按钮
- [ ] 音效（Web Audio API）：
  - 吞噬音效
  - 分裂音效
  - 加速音效
  - 死亡音效
- [ ] 视觉优化：
  - 蚯蚓颜色渐变（头部到尾部）
  - 泥洞特效
  - 道具发光/闪烁
  - 分裂抖动动画
- [ ] 性能优化：
  - 分段渲染优化（远处简化）
  - 碰撞检测优化（空间分割）
  - 分裂体数量上限
- [ ] 响应式：适配不同屏幕尺寸
- [ ] 代码整理：注释、模块化、设计文档

**验收标准：**
- 所有机制整合后稳定运行
- 无明显性能问题（60 FPS）
- 完整的开始-游戏-结束流程
- 代码结构清晰，可扩展

## 开发流程

```
┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│  开发阶段N   │ ──→ │  交付原型   │ ──→ │  用户反馈   │
│  (编码实现)  │     │ (HTML文件)  │     │ (调整建议)  │
└─────────────┘     └─────────────┘     └──────┬──────┘
                                               │
                                               ▼
                                        ┌─────────────┐
                                        │  调整优化   │
                                        │ (修改代码)  │
                                        └──────┬──────┘
                                               │
                                               ▼
                                        ┌─────────────┐
                                        │  确认通过   │ ──→ 进入阶段 N+1
                                        │  (验收OK)   │
                                        └─────────────┘
```

**每阶段交付后，请告诉我：**
1. 手感是否流畅？操控方式是否舒适？
2. 机制是否符合预期？有没有理解偏差？
3. 难度是否合适？
4. 有什么想调整或新增的地方？
