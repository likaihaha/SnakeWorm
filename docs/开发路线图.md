## 第二大阶段：Z字形关卡系统

> 设计文档：`docs/新玩法技术评估.md` + `docs/关卡设计玩法评估.md`

### 设计原则

> **不要做成纯线性关卡**，而是"开放世界 + 锁钥结构"。

- 地图保持 4000×3000 整体开放空间
- 关键节点放置条件门（藤蔓/岩石/能量墙）
- 门的条件透明（靠近时显示提示）
- 玩家可自由探索已解锁区域
- 通关后回访有隐藏内容

### Z字形路径

```
地图：5列×5行，每格 800×600，总计 4000×3000

路径：1→2→3→4→5↑10→9→8→7→6↑11→12→13→14→15↑20→19→18→17→16↑21→22→23→24→25

出生点：关卡1（左下角）
终点：关卡25（右上角）
```

### 难度分层

| 阶段 | 关卡 | 核心机制 |
|------|------|---------|
| 基础层 | 1~5 | 移动、吃宝珠、躲避障碍 |
| 战斗层 | 6~10 | 引入敌人，学习战斗 |
| 策略层 | 11~15 | 宝珠选择、路径规划、资源管理 |
| 技巧层 | 16~20 | 狭窄通道、移动障碍、限时挑战 |
| 终极层 | 21~25 | 综合考验，最终 Boss |

### 实现阶段

| Phase | 内容 | 依赖 |
|-------|------|------|
| Phase A | ZoneManager + 区域数据（纯数据层） | 无 |
| Phase B | Barrier 门系统（墙壁阻挡） | Phase A |
| Phase C | 宝珠/敌人按区域生成 | Phase A |
| Phase D | 通关条件 + 进度存储 | Phase B |
| Phase E | 自由回访 + 区域实体懒加载 | Phase C + D |

### 与饥饿系统的适配

- 关卡切换时重置饥饿计时器
- 每个关卡内保证宝珠充足

### 与敌人系统的适配

- 敌人锁定在当前关卡区域，不跨区追逐
- 子弹到达区域边界自动消失

### 与幼体系统的适配

- 幼体跟随玩家自动穿越已解锁区域
- 未解锁区域的门对幼体同样阻挡

## 设计红线

| ❌ 不要 | 原因 |
|--------|------|
| 做纯线性无回头路 | 卡关 = 死局 |
| 死亡重置所有进度 | 惩罚过重，玩家退出 |
| 关卡之间无差异 | 换色不叫关卡设计 |
| 把后代做成数值工具 | 情感变算计 |
| 让后代太多 | 1~3 只最佳，超过 5 只分不清 |
| 做复杂培养系统 | 不符合任天堂风格 |
| 后代可交易/丢弃 | 一旦可"处理"就不再是"家人" |
| 惩罚不养后代 | 后代应是锦上添花 |
