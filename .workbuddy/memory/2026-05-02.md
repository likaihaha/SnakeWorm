## editor 背景编辑器 Bug 修复
- **问题**: `canvas.js` 和 `keyboard.js` 调用了 `BackgroundEditor` 上不存在的方法，导致运行时报错
- **缺失方法（共 8 个）**: `getSelectedElements`、`getElement`、`getElementBounds`、`moveElement`、`selectAllElements`、`selectLayerByIndex`、`setTool`、`togglePreview`
- **修复**: 在 `core.js` 的 `BackgroundEditor` 类中补齐所有缺失方法
  - `getSelectedElements()` — 返回 `selectedElements` 数组
  - `getElement(id)` — 按 ID 获取元素（background/shape）
  - `getElementBounds(id)` — 委托给 `elementFactory.getElementBounds()`
  - `moveElement(id, dx, dy)` — 相对偏移移动元素（坐标+控制点）
  - `selectAllElements()` — 全选图层
  - `selectLayerByIndex(index)` — 数字键选图层
  - `setTool(tool)` — 工具切换（v/h/z/r/c/l/p）
  - `togglePreview()` — F5 预览模式
- **额外防御**: `index.html` 中 `toggleControlPointEditMode` 调用前加了 `typeof` 检查
- **根本原因**: `canvas.js`、`elements.js`、`keyboard.js` 等模块在开发时预期 `BackgroundEditor` 有这些方法，但 `core.js` 未实现

## v1.60 全身碰撞系统
- **需求**: 虫虫全身与敌人全身都能产生碰撞（不再只有头部vs头部），但只有虫虫头部撞到敌人时敌人才受伤
- **改动**:
  - `src/enemy.js` `checkCollisionWithPlayer()` 从只检测敌人头部vs虫虫头部 → 敌人全身vs虫虫全身双层循环
  - 敌人每段半径按 `size * (1 - j * ENEMY_TAIL_TAPER)` 计算（越后越小）
  - 返回被碰撞的虫虫段索引（0=头部，>0=身体）
  - `src/game.js` 玩家/AI成年体碰撞处判断 `hitSeg === 0` 时才双向伤害，>0 时仅虫虫扣血
- **规则**: 虫虫全身碰敌人都扣虫虫的血，但只有头部撞敌人才对敌人造成伤害
