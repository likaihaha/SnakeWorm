## v0.91 静止时身体正弦波波动动画

### 功能设计
- **静止时生命感波动**：虫虫没有移动时，身体从头到尾产生正弦波小幅度波动
- **波动方向**：垂直于虫虫当前朝向
- **波动幅度**：静止时 2.5 像素，移动时 0.3 像素（几乎不可见）
- **波动速度**：每秒 4 弧度（约 0.64 Hz）
- **波长参数**：段索引间距 0.6 弧度，形成从头到尾传递的波浪效果

### 实现细节
```javascript
// 在 Worm 构造函数中初始化
this.idleWavePhase = Math.random() * Math.PI * 2;

// 在 update 中更新相位
this.idleWavePhase += dt * 4;

// 在 draw 中应用偏移
const waveAmplitude = this.isMoving ? 0.3 : 2.5;
const v = this.velocity.normalize();
const perpX = -v.y;
const perpY = v.x;
const wave = Math.sin(this.idleWavePhase - i * 0.6) * waveAmplitude;
const seg = new Vector(segRaw.x + perpX * wave, segRaw.y + perpY * wave);
```

### 覆盖场景
- waitingForPlayer 等待玩家阶段：出场动画完成后继续波动
- AI 蚯蚓静止时：自动产生波动
- 玩家停止移动时：波动明显
- 玩家移动时：波动几乎消失（0.3px）

### 修复：头部抖动问题（三次修复）
1. **第一次修复**：头部（i=0）不参与正弦波波动
2. **第二次修复**：移除v0.18遗留的头部随机抖动代码

   - **问题**：每帧随机偏移导致头部在白圈内持续抖动

## Bug修复：TypeError Cannot read properties of undefined (reading 'push')

### 问题
- 控制台报错：`Uncaught TypeError: Cannot read properties of undefined (reading 'push')`
- 位置：`Worm.update` 第1940行
- 原因：`Worm.update` 中使用了 `this.foods.push(food)`，但 `foods` 数组属于 `Game` 类，不属于 `Worm` 类

### 解决方案
- 将 `this.foods.push(food)` 改为 `allFoods.push(food)`（使用传入的参数）
- 粒子特效改为暂存到 `worm.pendingParticles`，由 `Game.update` 统一收集到 `this.particles`

### 代码变更
```javascript
// 修改前（错误）
this.foods.push(food);
this.particles.push(new Particle(...));

// 修改后（正确）
allFoods.push(food);
if (!this.pendingParticles) this.pendingParticles = [];
this.pendingParticles.push(new Particle(...));

// Game.update中添加收集逻辑
if (worm.pendingParticles && worm.pendingParticles.length > 0) {
    for (const p of worm.pendingParticles) {
        this.particles.push(p);
    }
    worm.pendingParticles = [];
}
```

## v1.00 消耗尾巴保留初始速度修复

### 问题
v0.99的修复没有生效，尾巴仍然垂直落下。

### 根因
虫虫的`this.velocity`是归一化的单位向量（长度=1），保存这个单位向量再乘以0.5或0.3，得到的速度太小（0.5或0.3），几乎看不出效果。

### 修复
保存虫虫的**实际移动速度**（`velocity * speed`），而不是单位向量。

### 代码变更
```javascript
// 创建时保存实际移动速度
const actualVelocity = this.velocity.mult(this.speed);
this.shrinkingSegments.push({
    ...
    initialVelocity: actualVelocity.clone(),  // 保存实际速度（约5左右）
    ...
});

// 下沉时应用初始速度（比例调整）
const initialVx = shrinking.initialVelocity ? shrinking.initialVelocity.x : 0;
const initialVy = shrinking.initialVelocity ? shrinking.initialVelocity.y : 0;
shrinking.velocity = new Vector(
    initialVx * 0.8 + (Math.random() - 0.5) * 5,  // 水平：80%初始速度
    30 + Math.random() * 30 + initialVy * 0.5      // 垂直：下沉速度 + 50%初始垂直速度
);
```

- 虫虫向右移动时（速度约5），尾巴会明显向右飘落（水平速度约4）
- 虫虫向左移动时，尾巴会明显向左飘落

## v1.01 消耗尾巴保留初始速度修复（速度单位修正）

### 问题
v1.00的修复仍然不明显，尾巴左右飘落效果很弱。

### 根因
速度单位不匹配：
- 虫虫移动：`velocity * speed * dt * 60`（每秒移动 `velocity * speed * 60` 像素）
- 尾巴下沉：`velocity * dt`（每秒移动 `velocity` 像素）

所以保存的 `velocity * speed`（约5像素/秒）相对于下沉系统来说太小了。

### 修复
保存速度时乘以60：`velocity * speed * 60`（约300像素/秒），然后应用时降低比例：
- 水平：`initialVx * 0.1`（约30像素/秒，约0.5像素/帧）
- 垂直：`initialVy * 0.05`（约15像素/秒）

### 代码变更
```javascript
// 保存时乘以60
const actualVelocity = this.velocity.mult(this.speed * 60);

// 应用时降低比例
shrinking.velocity = new Vector(
    initialVx * 0.1 + (Math.random() - 0.5) * 5,
    30 + Math.random() * 30 + initialVy * 0.05
);
```

## v1.14 宝珠颜色鲜艳优化

### 问题
宝珠颜色不够鲜艳，被发光效果遮盖。

### 原因分析
1. Screen混合模式让颜色变亮变淡
2. 光环太多（最多4个）稀释颜色
3. 发光区域太大（2.2倍半径）

### 修复
1. **混合模式**：从screen改为source-over，颜色不再被稀释
2. **光环数量**：减少（最多2个），提高透明度(0.4)和线宽(1.5)
3. **发光区域**：缩小（从2.2倍改为1.6倍），添加中间颜色stop(0.7位置0.6透明度)
