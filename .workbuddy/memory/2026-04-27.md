  4. 虫虫又飞到左边 → 重置enterPhase=1导致动画重新从-50开始

- 最终修复：出场动画在waitingForPlayer期间已全部完成，鼠标进入白圈时直接isEntering=false

2026-04-27 03:10 - v0.43: 修复尸体碰底发射宝珠系统
- 问题：尸体下沉到框体底部，没有任何身体部分触发宝珠发射
- 修复3个bug：
  1. 【核心bug】DeadBody.update()内部设置hasEmitted=true后返回'emit'，外部调用处if (result==='emit' && !body.hasEmitted)中!body.hasEmitted永远为false，导致emitFoodFromBottom()永不执行。修复：外部条件改为if (result === 'emit')
  2. 【物理bug】DeadBody所有段共用单一this.velocity，每处理一段就加一次重力、阻尼一次，导致速度异常累积且段间不会自然分离。修复：改为每段独立的velocities数组
  3. 【运动bug】emitFoodFromBottom()创建的宝珠在底部（y=CANVAS_HEIGHT+20），但velocity.y=0，受重力影响会直接掉出屏幕。修复：给宝珠添加向上的初速度velocity.y = -(2.0 + Math.random() * 1.5)

2026-04-27 03:14 - v0.44: 修复底部发射宝珠被清理逻辑误删
- 用户反馈：宝珠在框体下面发射，碰到框体底部就消散了
- 原因：Game.update()中清理逻辑if (food.pos.y > CANVAS_HEIGHT - deathMargin)会把所有y大于底部死亡线的宝珠删除。底部发射的宝珠从y=CANVAS_HEIGHT+20向上飞，还没进入画面就被当成'掉出底部'删掉了
- 修复：清理条件增加food.velocity.y >= 0，只删除向下运动的宝珠；向上飞的允许穿过底部区域进入画面
