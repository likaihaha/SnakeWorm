# 版本号管理规范

## 版本号格式

采用 `主版本.次版本` 两位格式，例如 `1.51`。

- **主版本**：重大架构调整、核心玩法重构时递增（如 1.x → 2.0）
- **次版本**：每次功能新增、Bug 修复、体验优化后递增（如 1.50 → 1.51）

## 修改位置

**唯一需要手动修改的地方**：

```javascript
// src/config.js
export const CONFIG = {
    VERSION: '1.51',  // ← 只改这里
    // ...
};
```

页面中所有显示版本号的位置（`title`、页面标题、开始界面）均由 `src/main.js` 在初始化时自动从 `CONFIG.VERSION` 读取并写入，**不需要手动改 HTML**。

HTML 中对应位置已添加 `data-version` 属性：

```html
<span data-version style="color:#888; font-size:14px;">v1.51</span>
```

## 升级时机

| 场景 | 是否升级 | 示例 |
|------|---------|------|
| 新增功能 | ✅ 升级 | 新增调试面板按钮、新增观战模式 |
| Bug 修复 | ✅ 升级 | 修复敌人吃幼体时的 TypeError |
| 性能优化 | ✅ 升级 | 碰撞检测空间分区优化 |
| 纯文案/注释修改 | ❌ 不升 | 修改错别字、补充代码注释 |
| 临时调试验证 | ❌ 不升 | 本地测试参数调整 |

## 提交规范

升级版本号时，Git commit message 建议包含版本号：

```bash
git commit -m "feat: 死亡后进入观战模式 (v1.51)"
```

方便后续通过 `git log` 快速追溯每个版本对应的代码。
